## 📋 Overview
Building detection rules and dashboards in Kibana to detect the C2 activity generated during the Day 21 simulation — closing the loop between attack simulation and detection.

## 🎯 Key Steps
- Analyse the C2 beacon traffic in Kibana Discover
- Identify the Sysmon Event IDs generated by the attack simulation
- Build a detection rule for: suspicious outbound connections on non-standard ports
- Build a detection rule for: process spawning unexpected network connections
- Create a C2 Activity dashboard combining process, network, and auth events
- Tag rules with MITRE ATT&CK: T1071 (C2), T1059 (Execution)

## 🔍 KQL Detection Query
```kql
event.code: "3" and not destination.port: (80 or 443 or 53)
```
